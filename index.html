How CKY Works

The CKY algorithm fills the table bottom-up. Diagonal cells contain POS tags for each word. Upper cells combine constituents using grammar rules. The top-right cell contains the start symbol (S) if the sentence is grammatical.

Chomsky Normal Form Conversion

Why CNF?

CKY requires all grammar rules to be in Chomsky Normal Form: either A → BC (two non-terminals) or A → a (one terminal). The ATIS grammar expands from 5,517 to ~13,500 rules after CNF conversion due to binarization of longer rules.

Structural Ambiguity Analysis

"Flying planes can be dangerous"

Interpretation 1: Gerund Phrase

"Flying" is a verb (VBG) - the act of piloting planes

"The act of flying planes can be dangerous"

[NP [VP Flying planes]] can be dangerous ↑ "Flying" is the head

Interpretation 2: Adjective

"Flying" is an adjective (JJ) - modifying planes

"Planes that are flying can be dangerous"

[NP [JJ Flying] planes] can be dangerous ↑ "planes" is the head

Parser Limitation

Dependency parsers trained on treebanks can only output ONE interpretation. They cannot represent ambiguity - they pick the most probable parse based on training data. This is why human disambiguation often remains necessary for NLP systems.

CKY Algorithm Details

Implemented the Cocke-Kasami-Younger algorithm:

  • Grammar: ATIS CFG (5,517 productions)
  • CNF Conversion: Chomsky Normal Form
  • Back-pointers: Tree reconstruction
  • Multiple parses: Handles ambiguity
Time Complexity $$O(n^3 \cdot |G|)$$

where n = sentence length, |G| = grammar size

TF-IDF Vectorization

Custom implementation from scratch:

TF-IDF Weight $$\text{TF-IDF}(t, d) = \log_{10}(\text{count} + 1) \times \log_{10}\left(\frac{N}{df_t}\right)$$
  • Documents: 1,000 samples
  • Cosine Similarity: Semantic matching

PPMI Analysis

Positive Pointwise Mutual Information:

PPMI Formula $$\text{PPMI}(x, y) = \max\left(\log_2\frac{p(x,y)}{p(x) \cdot p(y)}, 0\right)$$
  • Word associations: Collocation detection
  • Co-occurrence: Context windows

LSTM Architecture

Input (100 tokens) -> Embedding (300D Word2Vec) -> LSTM(128, dropout=0.2) -> LSTM(64, dropout=0.2) -> LSTM(32, dropout=0.2) -> Dense(64, ReLU) -> Softmax(9 classes)
  • Dataset: CoNLL2003
  • Tags: BIO scheme (9 classes)

Entity Types

BIO tagging for 4 entity categories:

  • PER: Person names
  • ORG: Organizations
  • LOC: Locations
  • MISC: Miscellaneous entities
